    <h1 id="pacman">Pacman</h1>
    <p>Focues on esoteric usages I sometimes do, instead of re-searching</p>
    <blockquote>
      <p>Perhaps do a single packages for all (apk, pacman, apt)?</p>
    </blockquote>
    <p><strong>Purge Package</strong></p>
    <pre><code>pacman -Rns {package}
</code></pre>
    <p>The most common use case is to just delete a package {package}, removing all it's configurations and dependencies.</p>
    <ul>
      <li>R is the <em>Remove</em> operation</li>
      <li>n flag indicates to remove the package's configuration</li>
      <li>s flag indicates to remove unnecessary dependencies</li>
    </ul>
    <p><strong>List Orphans</strong></p>
    <pre><code>pacman -Qdt
</code></pre>
    <ul>
      <li>Q is the <em>Query</em> action</li>
      <li>d filters only depnedencies</li>
      <li>t filters only those who are unrequired</li>
    </ul>
    <p><strong>Delete orphans</strong></p>
    <pre><code>pacman -Rns $(pacman -Qdtq)
</code></pre>
    <p>Notice that we added a <em>q</em> option to the <em>List Orphans</em> command, it prints less information. Specifically, it makes it so the <em>Query</em> operation will only return the package name. That way, we can easily pass it as an argument to the <em>Remove</em> operation</p>
